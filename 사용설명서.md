## 1. 이게 뭐 하는 프로그램인가요?

**Langent(랭전트)** 은 내 컴퓨터에 흩어져 있는 문서들(메모, PDF, 보고서 등)을
**AI가 읽고 검색할 수 있는 "지식 창고"로 만들어주는** 무료 프로그램입니다.

> 📚 **비유:** 도서관 사서와 같습니다.
> 책(문서)을 잔뜩 던져주면, 사서가 **비슷한 주제끼리 정리**해 책장에 꽂고,
> "AI 관련 책 어디 있어요?"라고 물으면 **바로 찾아줍니다.**
> 게다가 그 책장을 **3D 우주 지도(별처럼 빛나는 점들)** 로 보여줍니다.

할 수 있는 일:
- **의미로 검색** — 단어가 똑같지 않아도 "비슷한 뜻"으로 문서를 찾습니다.
- **3D 지도 보기** — 내 문서들이 어떤 주제로 뭉쳐 있는지 한눈에 봅니다.
- **AI에게 내 문서 쓰게 하기** — Claude·Cursor 같은 AI 도구에 연결하면, AI가 내 문서를 "기억"처럼 참고해 답합니다. ("MCP" 연결 — 뒤에서 설명)

> 이 폴더는 인터넷에 공개된 원본 프로그램([AlexAI-MCP/langent](https://github.com/AlexAI-MCP/langent))을 **더블클릭만으로 쓰게** 해 주는 "리모컨(.bat 3개)"입니다.

## 2. 먼저 알아둘 단어들 (용어 사전)

| 단어 | 쉬운 뜻 |
|---|---|

| **.bat 파일** | 더블클릭하면 명령들을 자동 실행해 주는 윈도우 "자동 실행 파일". (예: `INSTALL.bat`) |
| **검은 창 / cmd / 콘솔** | 글자만 나오는 까만 창. 무서워 보여도 **진행 상황을 글로 보여줄 뿐**입니다. |

| **ChromaDB** | 그 숫자(벡터)들을 저장해 검색하게 해 주는 작은 데이터베이스. |
| **MCP** | AI 도구가 외부 프로그램(여기선 Langent)을 쓰게 해 주는 **연결 규약**. |

| **워크스페이스(workspace)** | 내가 AI에게 읽힐 **문서를 넣는 폴더**. |

## 4. 1단계 · Python 설치 (필요 프로그램)

> 🐍 Python은 이 프로그램의 "엔진"입니다. **딱 한 번만** 설치하면 됩니다.
> 이미 있으면 이 단계는 건너뛰어도 됩니다. (잘 모르겠으면 그냥 따라 하세요.)

### 따라 하기
1. 브라우저(엣지·크롬 등)에서 **https://www.python.org/downloads/** 접속
2. 큰 노란색 **"Download Python 3.xx.x"** 버튼 클릭 (3.10 이상이면 OK, 3.12 권장)
3. 내려받은 설치 파일을 **더블클릭**
4. ⚠️ **여기가 제일 중요!** 첫 화면 맨 아래
   ☑ **"Add python.exe to PATH"** 체크박스를 **반드시 체크**하세요.
   - 안 하면 나중에 `"Python 3.10 이상이 필요합니다"` 오류가 납니다.
   - 깜빡했다면? 설치 파일을 다시 실행해 체크하고 다시 설치.
5. **"Install Now"** 클릭 → 끝날 때까지 대기 (1~2분)
6. **"Setup was successful"** 이 뜨면 **Close**

### Git도 설치 (소스 내려받기용)
1. **https://git-scm.com/download/win** 접속 → 설치 파일 자동 다운로드
2. 더블클릭 → 옵션은 **기본값 그대로 Next** 반복 → Install → Finish

✅ 확인(선택): Windows 키 → `cmd` → Enter → `python --version` 입력 → `Python 3.xx.x` 가 나오면 성공. (안 나오면 `py -3 --version` 도 시도)

## 6. 3단계 · 설치하기 (INSTALL.bat)

> 🎯 **딱 한 번만** 합니다. 한 번 설치하면 다음부터는 [7단계 RUN.bat](#7-4단계--문서-넣고-실행하기-runbat)만 하면 됩니다.

### 따라 하기
1. 키트 폴더를 엽니다. `INSTALL.bat`, `RUN.bat`, `UNINSTALL.bat`이 보입니다.
2. **`INSTALL.bat`** 더블클릭
3. 까만 창이 뜨며 6단계로 진행됩니다. (이 창은 정상입니다. 끄지 마세요. **5~15분** 걸립니다.)
   - `[1/6] Python 확인` → 파이썬 찾기 (`python`→`python3`→`py -3` 순서로 자동)
   - `[2/6] 소스코드 확인` → GitHub에서 langent 내려받기(clone)
   - `[3/6] 가상환경 생성` → 전용 상자(venv) 만들기
   - `[4/6] 의존성 설치` → 필요한 부품 설치 (**여기가 가장 오래 걸립니다. 기다리세요.**)
   - `[5/6] 환경 설정` → `.env`·`workspace`·`data` 폴더 만들기 (예시 문서도 자동 복사)
   - `[6/6] MCP 설정 생성` → `mcp_config_py.json`·`mcp_config_npx.json` 만들기
4. **"설치 완료!"** 가 뜨면 성공입니다.

### ⚠️ 자주 나오는 설치 오류
- `[ERROR] Python 3.10 이상이 필요합니다!` → Python을 **"Add to PATH" 체크**하고 (재)설치. 이미 설치했다면 **cmd 창을 새로 열어** 다시 시도.
- `[ERROR] langent 폴더가 없고 Git도 미설치!` → Git을 설치(https://git-scm.com/download/win)한 뒤 다시 실행.
- `[ERROR] 설치 실패!` → ① 인터넷 확인 ② `python -m pip install --upgrade pip` ③ 그래도 안 되면 Python 3.12 또는 Visual C++ Build Tools 설치.

> 🏢 보안 경고("이 앱이 변경하도록 허용?")가 뜨면 **"예"** 를 누르면 됩니다. 안전합니다.

## 7. 4단계 · 문서 넣고 실행하기 (RUN.bat)

### 7-1. 먼저 문서를 넣으세요
설치하면 생긴 **`workspace`** 폴더에 AI에게 읽힐 문서를 넣습니다.

| 종류 | 확장자 |
|---|---|
| 마크다운 | `.md` |
| 텍스트 | `.txt` |
| PDF | `.pdf` |
| 표 데이터 | `.csv` |
| 구조화 데이터 | `.json` |
| 설정/데이터 | `.yaml` |

> 하위 폴더 안의 파일도 자동으로 찾아 읽습니다. (설치 직후엔 예시 문서가 들어 있어 바로 연습 가능)

### 7-2. RUN.bat 실행
**`RUN.bat`** 더블클릭 → 아래 메뉴가 나옵니다:

```
============================================================
  Langent v3 - Personal Ontology + 3D Knowledge Nebula
============================================================

    [1] Nebula 서버 시작      http://localhost:8000
    [2] 문서 인제스트         workspace 폴더 스캔
    [3] MCP 서버 시작         MCP 클라이언트 연동
    [4] 지식 검색 (CLI)       벡터 DB 시맨틱 검색
    [5] 시스템 상태           버전, 패키지 확인
    [6] GitHub 업데이트       최신 소스 + 의존성 갱신
    [7] 테스트 실행           pytest + ruff + mypy
    [8] 삭제 (Uninstall)      venv, data 등 정리
    [0] 종료

============================================================
  선택 (0~8):
```

👉 **번호를 누르고 Enter** 하면 됩니다.

### 7-3. 처음이라면 이 순서로
1. **`2`** Enter → 문서 인제스트 (workspace 문서를 읽어 등록 + 관계 연결)
2. **`1`** Enter → Nebula 서버 시작 → 브라우저에 3D 우주 화면이 열림 (`http://localhost:8000`)
3. 다 봤으면 검은 창에서 **Ctrl+C** → 메뉴로 돌아옴 → `0`으로 종료

> 💡 `2`(인제스트)를 먼저 해야 검색·3D 화면에 내용이 보입니다. 문서를 새로 추가하면 다시 `2`를 돌리세요.

## 8. 메뉴 0번부터 8번까지 (하나도 빠짐없이)

### 🔢 1 · Nebula 서버 시작
- **무엇:** 3D 우주(별) 화면을 브라우저에서 봅니다. `http://localhost:8000`이 자동으로 열립니다.
- **조작:** 별 클릭=문서 보기 / 왼쪽드래그=회전 / 휠=확대·축소 / 상단 검색바+Enter=관련 문서 빛남.
- **끄기:** 검은 창에서 **Ctrl+C** → 메뉴 복귀.
- API 문서 화면도 있음: `http://localhost:8000/docs`

### 🔢 2 · 문서 인제스트
- **무엇:** `workspace` 폴더의 문서를 읽어 검색 가능하게 만들고(인제스트), 문서 간 관계도 연결(link)합니다.

## 10. AI 도구에 연결하기 (MCP 설정)

설치하면 폴더에 **`mcp_config_py.json`**(권장)과 **`mcp_config_npx.json`** 두 파일이 생깁니다.
이 파일을 **메모장으로 열어** 안의 `"langent": { ... }` 부분을, 쓰는 AI 도구의 설정 안 `"mcpServers": { }` 에 붙여넣으면 연결됩니다. (경로는 자동으로 채워져 있습니다.)

### Claude Desktop
1. Claude Desktop → **Settings → Developer → Edit Config** (또는 `%APPDATA%\Claude\claude_desktop_config.json`)
2. `"langent": { ... }` 를 `"mcpServers": { }` 안에 추가
3. Claude Desktop **완전 종료**(트레이 아이콘 우클릭 → 종료) 후 다시 실행

### Google Antigravity
1. 채팅 패널 → **... → MCP Servers → Manage MCP Servers → View raw config** (또는 `%USERPROFILE%\.gemini\antigravity\mcp_config.json`)
2. `"langent": { ... }` 추가 → **Refresh**

### Claude Code (터미널)
```
claude mcp add langent -- "내경로\venv\Scripts\python.exe" -m langent.server.mcp_server
```
> 정확한 경로는 `mcp_config_py.json`을 메모장으로 열어 확인.

### Cursor
1. **Settings → MCP → Add new MCP server**
2. `mcp_config_py.json` 내용 참고하여 추가

### 이미 다른 MCP 서버가 있다면
기존 항목 뒤에 **쉼표(`,`)** 를 넣고 langent를 추가하세요:
```json
{
  "mcpServers": {
    "기존서버": { ... },
    "langent": { ... }
  }
}
```

> ⚠️ JSON에서 경로의 역슬래시는 `\\`(두 개)로 써야 합니다. 자동 생성된 파일을 그대로 복사하면 이미 맞게 돼 있습니다.

### 연결 후
AI에게 이렇게 말하면 됩니다:
- "내 문서에서 'AI 전략'에 대한 내용을 찾아줘"
- "워크스페이스의 새 문서를 읽어들여줘"
- "내 지식 베이스 상태를 확인해줘"

## 11. 파일과 문서가 어디에 있나요? (위치 총정리)

### 처음부터 있는 파일 (이 키트)
```
Langent-MCP_One-Click_Kit\
├── INSTALL.bat       ← 맨 처음 한 번 (설치)
├── RUN.bat           ← 매번 실행 (메인)
├── UNINSTALL.bat     ← 깔끔하게 삭제
├── README.md         ← 요약·참조 설명서
└── 사용설명서.md      ← 지금 보는 이 문서 (왕초보 완전판)
```

### 설치하면 자동으로 생기는 것
```
├── mcp_config_py.json   ← MCP 설정 (Python 방식, 권장)
├── mcp_config_npx.json  ← MCP 설정 (npx 방식)
├── langent/             ← 프로그램 소스코드
│   ├── .env             ← 설정 파일
│   └── samples/         ← 예시 문서
├── venv/                ← 가상환경 (전용 부품 상자)
├── workspace/           ← 여기에 내 문서 넣기
└── data/chroma_db/      ← 벡터 데이터베이스 (검색 데이터)
```

### .env 설정 파일 (보통 손댈 필요 없음)
`langent\.env`에 자동 생성됩니다. 문서 폴더를 바꾸거나 포트를 바꿀 때만 수정.

| 항목 | 뜻 | 기본값 |
|---|---|---|
| `LANGENT_WORKSPACE` | 읽을 문서 폴더 | 설치폴더\workspace |
| `API_PORT` | 3D 서버 포트 | 8000 |
| `CHROMA_DB_PATH` | 검색 데이터 위치 | 설치폴더\data\chroma_db |
| `NEO4J_PASSWORD` | Neo4j 비밀번호 | (Neo4j 쓸 때만) |
| `LLM_MODE` | LLM 모드 | fake |

> ⚠️ **.env 작성 규칙:** 값에 **따옴표(`"`)를 넣지 마세요.** (`API_PORT=8000` O / `API_PORT="8000"` X). 따옴표가 들어가면 `python-dotenv could not parse` 오류가 납니다.
> 🔐 `.env`는 비밀번호가 들어갈 수 있으니 **남과 공유하거나 인터넷에 올리지 마세요.**

## 12. 안전과 개인정보

- ✅ **내 컴퓨터 안에서 동작합니다.** 문서는 로컬 벡터 DB(`data\chroma_db`)에 저장됩니다. 기본 설정(`LLM_MODE=fake`)은 외부 AI를 부르지 않고도 검색·3D가 됩니다.
- ✅ **시스템을 건드리지 않습니다.** 가상환경(venv)에 따로 설치되어, 내 컴퓨터의 다른 Python·프로그램에 영향이 없습니다.
- ✅ **삭제가 깔끔합니다.** UNINSTALL.bat은 venv·소스·데이터만 지우고 시스템 Python·Git·환경변수는 그대로 둡니다.
- ⚠️ **AI 도구(MCP)에 연결하면** 그 AI 도구가 내 문서 내용을 읽어 답에 사용할 수 있습니다. 민감한 문서는 넣기 전에 한 번 생각하세요.
- ⚠️ **`.env` 파일**에 Neo4j 비밀번호 등이 들어갈 수 있으니 공유 금지.

## 13. 문제가 생겼을 때 (오류 대처 총정리)

| 화면 메시지 / 증상 | 원인 | 해결 |
|---|---|---|
| `Python 3.10 이상이 필요합니다` | Python 미설치 또는 PATH 미등록 | Python을 **"Add to PATH" 체크**하고 (재)설치. 이미 했다면 cmd 새로 열기 |
| `Git not found` / `Git도 미설치` | Git 미설치 | https://git-scm.com/download/win 설치 후 재실행 |
| `가상환경이 없습니다` | INSTALL.bat 안 함/실패 | `INSTALL.bat` 먼저 실행 |
| `설치 실패!` (pip) | 인터넷 끊김/패키지 비호환 | 인터넷 확인 → `python -m pip install --upgrade pip` → Python 3.12 시도 |
| `python-dotenv could not parse` | `.env` 값에 따옴표(`"`) 있음 | `.env` 열어 모든 따옴표 제거 후 저장 |
| `Workspace: .` (경로 이상) | `.env` 파싱 실패 | 위와 동일 — 따옴표 제거 |
| `Neo4j connection failed` | Neo4j 미설치 | **정상** — 무시해도 됨 |
| `http://localhost:8000` 안 열림 | 서버가 안 돌고 있음 | `RUN.bat → [1]`로 서버를 켰는지 확인 |
| 포트 충돌 (8000 사용 중) | 다른 프로그램이 8000 사용 | `.env`에서 `API_PORT=8001` 등으로 변경 |
| `Found 0 files` | workspace가 비어 있음 | workspace에 문서 넣고 `[2]` 다시 |
| AI 도구에서 Langent 안 보임 | MCP 설정 오류 | `mcp_config_py.json` 경로 확인(`\`→`\\`) + AI 도구 **재시작** |
| `langent` 명령어 못 찾음 | 가상환경 미활성 | RUN.bat으로 실행하면 자동 활성화됨 |
| 까만 창이 빨리 닫힘 | (드물게) | cmd를 먼저 연 뒤 `.bat`을 그 안에 드래그해 실행하면 메시지를 끝까지 볼 수 있음 |

## 14. 자주 묻는 질문 (FAQ)

**Q. 개발을 전혀 몰라도 되나요?**
A. 네. 번호 누르고 Enter만 하면 됩니다.

**Q. 제 문서가 외부로 새나가나요?**
A. 기본은 내 컴퓨터 안에서만 동작합니다. 단, **AI 도구(MCP)에 연결하면** 그 AI가 문서를 읽어 답에 씁니다.

**Q. 인터넷이 꼭 필요한가요?**
A. **설치·업데이트** 때만요. 그 뒤 검색·3D 보기는 인터넷 없이도 됩니다.

**Q. 돈이 드나요?**
A. 무료입니다. 이 키트와 원본 Langent 모두 **Apache License 2.0** 오픈소스입니다. (상업적 사용도 허용 — 라이선스 고지만 유지)

**Q. Mac이나 휴대폰에서도 되나요?**
A. 이 키트(.bat)는 **윈도우 전용**입니다. (Mac/Linux는 원본 소스를 명령어로 직접 설치해야 합니다.)

**Q. Neo4j를 꼭 깔아야 하나요?**
A. 아니요. `Neo4j connection failed`는 정상이며, 없어도 검색·3D·AI 연결이 다 됩니다.

**Q. 문서를 추가하면 자동으로 반영되나요?**
A. 아니요. 추가 후 `RUN.bat → [2]`(인제스트)를 다시 돌려야 반영됩니다.

**Q. 메뉴 [7] 테스트는 꼭 해야 하나요?**
A. 아니요. 개발자용입니다. 일반 사용자는 안 해도 됩니다.
